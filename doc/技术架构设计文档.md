# DownloadApp 技术架构设计文档

## 1. 需求分析

### 1.1 功能需求概述

根据PRD文档，DownloadApp需要实现以下核心功能：

| 功能模块 | 子功能 | 需求来源 |
| :--- | :--- | :--- |
| **下载管理** | HTTP/HTTPS下载、暂停/取消/断点续传、多任务下载、速度限制 | PRD §1.1 |
| **下载优化** | 智能重试机制、下载路径自定义、批量操作 | PRD §1.2 |
| **文件夹分享** | 本地文件夹浏览、文件夹共享、访问权限控制 | PRD §2.1 |
| **相册分享** | 相册图片/视频访问、选择分享、分享链接生成 | PRD §2.2 |

### 1.2 非功能性需求

| 类型 | 要求 | 需求来源 |
| :--- | :--- | :--- |
| **性能** | 下载速度不低于原生浏览器、启动时间<2秒、内存占用合理 | PRD §6.1 |
| **安全** | 用户数据加密存储、HTTPS传输、权限最小化 | PRD §6.2 |
| **兼容性** | 遵循Apple HIG、Dark Mode支持、适配不同屏幕尺寸 | PRD §6.3 |

## 2. 技术选型

### 2.1 核心技术栈

| 层级 | 技术 | 版本要求 | 选型理由 |
| :--- | :--- | :--- | :--- |
| **前端框架** | SwiftUI | iOS 16+, macOS 13+ | 官方跨平台UI框架，原生性能，开发效率高 |
| **语言** | Swift | 5.7+ | Apple生态首选语言，性能优秀 |
| **网络层** | URLSession | Apple原生 | 稳定可靠，支持断点续传，无需第三方 |
| **数据库** | Core Data | Apple原生 | Apple官方数据持久化框架，支持对象关系映射，与SwiftUI集成良好 |
| **图片缓存** | 自研实现 | 自研 | 基于NSCache和FileManager，轻量级实现 |
| **压缩解压** | 自研实现 | 自研 | 基于Compression框架，纯Swift实现 |

### 2.2 架构风格

采用 **分层架构 (Layered Architecture)** 结合 **MVVM设计模式**：

- **分层设计**：UI层 → 业务逻辑层 → 数据访问层 → 基础设施层
- **MVVM模式**：View → ViewModel → Model，实现UI与业务逻辑解耦

### 3.3 核心流程图

#### 3.3.1 下载任务流程

```
用户发起下载
    │
    ▼
创建 DownloadTask
    │
    ▼
检查本地缓存 → 有缓存则直接恢复断点
    │
    ▼
创建 URLSessionDownloadTask
    │
    ▼
开始下载 → 实时更新进度
    │
    ├──暂停请求 → 保存断点信息 → 暂停任务
    │
    ├──取消请求 → 删除临时文件 → 移除任务
    │
    └──下载完成 → 校验文件 → 移动到目标路径 → 更新数据库
```

#### 3.3.2 文件分享流程

```
用户选择文件/文件夹
    │
    ▼
检查访问权限
    │
    ├──有权限 → 生成分享链接
    │
    └──无权限 → 请求权限
    │
    ▼
设置分享有效期/权限
    │
    ▼
保存分享记录到数据库
    │
    ▼
返回分享链接给用户
```

## 8. 安全性考虑

### 8.1 数据安全

| 风险点 | 解决方案 |
| :--- | :--- |
| 用户数据泄露 | 使用Core Data与File Protection配合，敏感字段使用AES加密 |
| 网络传输安全 | 强制使用HTTPS，验证SSL证书 |
| 权限滥用 | 遵循权限最小化原则，仅请求必要权限 |

### 8.2 下载安全

| 风险点 | 解决方案 |
| :--- | :--- |
| 恶意文件下载 | 文件类型白名单校验，限制可下载文件类型 |
| 下载中断攻击 | 实现智能重试机制，数据完整性校验 |
| 资源耗尽攻击 | 限制最大并发任务数，实现速度限制 |

### 8.3 分享安全

| 风险点 | 解决方案 |
| :--- | :--- |
| 未授权访问 | 使用UUID+Token双重验证机制 |
| 链接泄露 | 设置分享有效期，支持手动撤销 |
| 权限越权 | 严格权限控制，区分只读/读写权限 |

## 9. 性能优化策略

### 9.1 下载性能

- **断点续传**：利用URLSession的resumeData实现断点续传
- **并发控制**：限制最大并发任务数，避免资源竞争
- **速度限制**：实现流量控制算法，避免带宽占用过高

### 9.2 内存优化

- **懒加载**：按需加载数据，避免一次性加载大量数据
- **图片缓存**：使用Kingfisher进行图片缓存和内存管理
- **及时释放**：任务完成后及时释放资源

### 9.3 启动优化

- **异步初始化**：将非关键初始化移到后台线程
- **按需加载**：避免启动时加载所有模块
- **缓存预热**：提前缓存常用数据

## 10. 代码安全性

### 10.1 输入验证

- 对所有用户输入进行严格验证
- 使用正则表达式校验URL格式
- 限制文件路径长度和字符范围

- 实现全局错误处理机制

- 使用统一的日志框架
- 分级日志（Debug/Info/Warning/Error）
- 生产环境禁用Debug日志
